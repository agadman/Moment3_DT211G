Utvärdering av mitt arbete med SASS

Fördelar

Att arbeta med SASS har varit en lärorik och intressant upplevelse. En av de största fördelarna jag märkt är hur mycket mer organiserad min CSS-kod blir. Genom att använda variabler, mixins och @use/@import kan jag återanvända kod och göra ändringar snabbt, vilket sparar både tid och energi. Strukturen blir tydligare, och det blir enklare att hantera större projekt.

En annan stor fördel är nestning, som gör att jag kan skriva mer läsbar kod och undvika upprepningar. Det känns naturligare att gruppera relaterade stilar på detta sätt.

Nackdelar

Nackdelarna är få, men en utmaning i början var att förstå hur allt hängde ihop, särskilt med filstrukturen och hur man bäst delar upp koden i olika partials. Dessutom kräver SASS en kompilering till vanlig CSS, vilket kan kännas som ett extra steg jämfört med att direkt skriva i CSS.

Jag märkte också att jag ofta fick error när jag höll på att skriva koden men ännu inte hade skrivit klart ett semikolon. Då fick jag ett felmeddelande om att ett semikolon saknades, och även om jag lade till det efteråt, hjälpte det inte alltid. Jag var tvungen att köra om byggprocessen genom npm start för att felet skulle försvinna.

Sammanfattning

Sammanfattningsvis tycker jag att SASS är ett kraftfullt verktyg som gör det enklare att hantera och underhålla CSS i större projekt. Trots en liten inlärningströskel känns det som en värdefull förbättring jämfört med vanlig CSS. Ett irritationsmoment var dock att små misstag, som att tillfälligt sakna ett semikolon, kunde orsaka fel som krävde att jag startade om byggprocessen för att lösa dem.
