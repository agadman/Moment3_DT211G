Utvärdering av mitt arbete med SASS

Fördelar

Att jobba med SASS har varit både lärorikt och intressant. En av de största fördelarna är hur mycket mer strukturerad min CSS-kod blir. Med variabler, mixins och @use/@import kan jag återanvända kod och snabbt göra ändringar, vilket sparar tid och underlättar arbetet. Koden blir mer överskådlig och hanterbar, särskilt i större projekt.

En stor fördel är möjligheten att använda nestning, vilket gör koden mer läsbar och minskar upprepningar. Det känns dessutom mer logiskt att samla relaterade stilar på detta sätt.

Nackdelar

En utmaning i början var att förstå hur allt hängde ihop, särskilt med filstrukturen och hur man bäst delar upp koden i olika partials. Dessutom kräver SASS en kompilering till vanlig CSS, vilket kan kännas som ett extra steg jämfört med att direkt skriva i CSS.

Jag märkte också att jag ofta fick error när jag höll på att skriva koden men ännu inte hade skrivit klart ett semikolon. Då fick jag ett felmeddelande om att ett semikolon saknades, och även om jag lade till det efteråt, hjälpte det inte alltid. Jag var tvungen att köra om byggprocessen genom npm start för att felet skulle försvinna.

Hur var det att komma igång att arbeta med en CSS-preprocessor?

Att börja med SASS var lärorikt men innebar en liten inlärningströskel. Det tog ett tag att förstå filstrukturen och hur @use fungerar. Små syntaxfel, som ett saknat semikolon, kunde orsaka kompileringsfel som ibland krävde en omstart av byggprocessen.

Trots detta gav SASS snabbt bättre struktur och återanvändbar kod. Variabler, mixins och nestning gjorde arbetet smidigare och mer effektivt. När jag väl kom in i det kändes det som en klar förbättring jämfört med vanlig CSS.

Sammanfattning

Sammanfattningsvis tycker jag att SASS är ett kraftfullt verktyg som gör det enklare att hantera och underhålla CSS i större projekt. Trots en liten inlärningströskel känns det som en värdefull förbättring jämfört med vanlig CSS. Ett irritationsmoment var dock att små misstag, som att tillfälligt sakna ett semikolon, kunde orsaka fel som krävde att jag startade om byggprocessen för att lösa dem.
